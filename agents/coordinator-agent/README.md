# 协调智能体 (Coordinator Agent)

**版本：** v0.1  
**创建时间：** 2026-03-23  
**职责：** 任务解析、子任务分解、智能体调度、结果汇总

---

## 一、核心职责

### 1.1 任务解析
- 理解用户指令的真实意图
- 识别任务类型（文章生成/代码审查/信息搜索等）
- 提取关键参数（主题、格式、发布渠道等）

### 1.2 子任务分解
- 将复杂任务拆解为可执行的子任务
- 确定子任务间的依赖关系
- 制定执行顺序（并行/串行）

### 1.3 智能体调度
- 根据子任务类型分配合适的智能体
- 监控执行进度
- 处理异常和重试

### 1.4 结果汇总
- 收集各智能体的执行结果
- 整合为最终输出
- 返回给用户

---

## 二、工作流程

```
用户指令
    ↓
┌──────────────┐
│ 协调智能体    │
│ 1. 解析意图   │
│ 2. 拆解任务   │
│ 3. 分配智能体 │
│ 4. 汇总结果   │
└──────┬───────┘
       │
       ├──→ 🔍 搜索智能体
       ├──→ ✍️ 内容智能体
       ├──→ 💻 代码智能体
       ├──→ 🎓 专家智能体
       ├──→ ✅ 审核智能体
       └──→ 📤 发布智能体
```

---

## 三、任务类型识别

| 任务类型 | 关键词 | 参与智能体 |
|---------|--------|-----------|
| 文章生成 | "写文章"/"生成报告" | 搜索→内容→专家→审核→发布 |
| 代码审查 | "检查代码"/"代码质量" | 代码→专家 |
| 信息搜索 | "搜索"/"查找" | 搜索→内容 |
| 数据分析 | "分析"/"统计" | 搜索→专家→内容 |
| 发布任务 | "发布"/"推送" | 审核→发布 |

## 四、API 设计

### 4.1 任务提交

```python
POST /api/v1/coordinator/submit
{
  "instruction": "帮我写一篇量子计算的科普文章",
  "priority": "normal",
  "deadline": "2026-03-23T12:00:00Z"
}
```

### 4.2 任务状态查询

```python
GET /api/v1/coordinator/task/{task_id}/status
```

### 4.3 任务取消

```python
POST /api/v1/coordinator/task/{task_id}/cancel
```
