# Article Writing Workflow v2.0 - 多智能体协同工作流

**版本：** v2.0  
**更新时间：** 2026-03-27  
**模型：** MiniMax-M2.7  
**许可证：** MIT

---

## 核心设计原则

1. **文本与图片完全分离** — 写作智能体和图片智能体并行工作，互不阻塞
2. **图片自主生成** — 封面图等使用 DALL-E 生成，摆脱依赖外部URL
3. **四维 QC 把关** — 合规安全 + 文本质量 + 图片质量 + 格式正确性

## 各智能体详细设计

### 1. CEO 决策节点

**输入：** 用户指令（"写一篇关于xxx的科普文章"）  
**输出：** 任务分配给各智能体  
**职责：**
- 确认选题和受众
- 分配模型（MiniMax-M2.7）
- 设置质量标准
- 终稿审批发布

---

### 2. 文本写作智能体 TextWriter

**模型：** MiniMax-M2.7  
**输入：** 选题 + 受众 + 字数要求  
**输出：** Markdown 格式文章草稿

**工作流程：**
```
接收写作指令
   ↓
搜集素材（情报搜索 via SearXNG）
   ↓
撰写初稿（结构：摘要 → 正文 → 结语）
   ↓
自检（长度、逻辑、AI痕迹）
   ↓
输出给 Integrator
```

**写作标准：**
- 口语化，避免生硬科普腔
- 每篇 1500-3000 字
- 段落不超过 4 句
- 禁用词：「首先」「其次」「综上所述」（套话去除）
- 专业术语加括号解释
- 具体案例和数据支撑

**禁止写入的内容：**
- 涉及政治敏感话题
- 未经验证的科学声称
- 侵犯隐私的个人信息
- 抄袭/洗稿内容

---

### 3. 图片智能体 ImageAgent

**模型：** MiniMax-M2.7  
**输入：** 文章主题 + 图片需求列表  
**输出：** 每张图片的 URL/本地路径 + 说明文字

**工作流程：**
```
接收图片需求
   ↓
封面图 → DALL-E 生成（必须）
   ↓
内容配图 → 优先级：官方机构 > 知识库 > 搜索引擎
   ↓
多模态验证（MiniMax-M2.7 判断相关性 ≥7分）
   ↓
输出：图片URL + alt文字 + 放置位置
```

**图片来源优先级：**
| 优先级 | 来源 | 说明 |
|--------|------|------|
| 1️⃣ | DALL-E 生成 | 封面图、分镜图、示意图 |
| 2️⃣ | NASA/EHT/ESO/官方 | 科学事实类图片 |
| 3️⃣ | Wikimedia Commons | Wikipedia 同源图 |
| 4️⃣ | Unsplash/Pexels | 通用配图 |

**图片生成提示词模板：**
```
封面图：[文章主题]，专业科普风格，高对比度，居中构图，4K
分镜图：[具体概念]示意图，简洁清晰，无文字，学术风格
```

**验证标准：**
- 相关性评分 ≥ 7/10（MiniMax-M2.7 多模态判断）
- 下载成功率 ≥ 95%
- 版权合规（CC协议/Official/AI生成）

---

### 4. 整合智能体 Integrator

**模型：** MiniMax-M2.7  
**输入：** 文本草稿 + 图片方案  
**输出：** 带图片的完整 Markdown 文章

**整合规则：**
- 每 500 字放置 1 张图片
- 封面图固定在文章开头
- 图片居中，alt 文字精确描述
- 微信公众号格式：标题用 ##，加粗用 **

---

### 5. QC 智能体 QCAgent v2.0

**模型：** MiniMax-M2.7  
**输入：** 完整文章草稿  
**输出：** 评分报告 + 修改建议

**四维评分体系（每项 0-10，总分 40）：**

| 维度 | 检查内容 | 权重 |
|------|---------|------|
| 合规安全 | 政治敏感词、隐私侵权、科学准确性、版权合规 | 25% |
| 文本质量 | 逻辑连贯性、可读性、原创性、套话去除、AI痕迹 | 30% |
| 图片质量 | 相关性、版权、清晰度、alt文字准确性 | 25% |
| 格式正确 | Markdown 格式、标题层级、字数、平台兼容性 | 20% |

**评分标准：**
- ✅ **通过**：总分 ≥ 32，且每维 ≥ 6
- 🔄 **修改**：总分 24-31，或某维 < 6 → 返回对应智能体修改
- ❌ **拒绝**：总分 < 24，或合规维 < 4 → 重新选题

**合规检查清单：**
- [ ] 无政治敏感内容
- [ ] 无未经证实科学声称
- [ ] 无个人隐私信息
- [ ] 图片版权合规或已注明来源
- [ ] 无抄袭/过度引用
- [ ] 事实陈述有可靠来源

**文本质量检查清单：**
- [ ] 阅读流畅，无语法错误
- [ ] 无"首先/其次/最后"套话序列
- [ ] 无重复段落或车轱辘话
- [ ] 术语解释充分
- [ ] 有具体案例或数据
- [ ] AI 痕迹检测（ai-humanizer-cn 通过）

**图片质量检查清单：**
- [ ] 每张图片与内容相关
- [ ] alt 文字描述准确
- [ ] 版权信息已标注
- [ ] 清晰度适合平台展示
- [ ] 封面图吸引力达标

**格式检查清单：**
- [ ] 字数在要求范围内
- [ ] 标题层级清晰（## / ###）
- [ ] 微信公众号可直接粘贴
- [ ] 无损坏图片链接
- [ ] 代码块/引用块格式正确

## 工作流执行示例

```
用户："写一篇关于黑洞探测的科普文章，公众号用"
   ↓
CEO：分配任务，MiniMax-M2.7
   ↓
TextWriter（并行）+ ImageAgent（并行）
   ↓
TextWriter → 初稿 + 自检报告
ImageAgent → 封面DALL-E + 3张配图 + 验证报告
   ↓
Integrator → 完整文章草稿
   ↓
QCAgent v2.0 → 四维评分
   ↓
≥32分 → 发布
<32分 → 返回对应智能体修改 → 重新QC（最多3轮）
   ↓
终稿发布 → /root/.openclaw/workspace/articles/
```

---

## 多轮迭代机制

| 轮次 | 通过标准 | 失败处理 |
|------|---------|---------|
| 第1轮 | 总分≥32 | 返回写作/图片/整合 |
| 第2轮 | 总分≥32 | 返回写作 |
| 第3轮 | 总分≥32 | ❌ 选题重做 |

每次 QC 返回时，明确指出：
- 哪一维不达标（具体到检查项）
- 扣分原因
- 修改建议
