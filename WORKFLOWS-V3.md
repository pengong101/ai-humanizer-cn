## 一、技能研发工作流

### 核心角色

| 角色 | 职责 |
|------|------|
| **main** | 布置任务 + 最终汇报 |
| **ResearchAgent** | 调研现有方案 + 技术方案对比 |
| **CodeAgent** | 开发实现 + 测试 + 文档 |
| **ReviewAgent** | 质量审核（创新性≥9，安全≥9） |

### 流程图

```
用户需求
    │
    ▼
main（布置任务）
    │
    ├───────────────────────────────────────┐
    │                                       │
    ▼                                       ▼
ResearchAgent ◄──────────────────► CodeAgent
    │                                    │
    │ 调研方案 ──────────────────────── 申请技术建议
    │ 对比分析 ◄────────────────────── 提供实现思路
    │                                    │
    └──────── 迭代直到 CodeAgent 确认方案 ─┘
                      │
                      ▼
               CodeAgent 开发
                      │
                      ├─→ 需要验证 ───→ ResearchAgent 核查
                      │
                      └─→ 完成开发
                              │
                              ▼
                        ReviewAgent 审核
                              │
                     ┌────────┴────────┐
                     ▼                 ▼
                  通过              不通过（修改，最多3轮）
                     │                 │
                     ▼          3轮未过 → main 决策
                 CodeAgent 发布
                     │
                     ▼
                main 汇报用户
```

## 二、雷达日报生成工作流

### 核心角色

| 角色 | 职责 |
|------|------|
| **main** | 布置任务 + 最终汇报 |
| **ResearchAgent** | 搜集雷达相关情报 + 事实核查 |
| **ContentAgent** | 生成日报 + 配图 + 格式优化 |
| **ReviewAgent** | 质量审核（可选，快速审核） |

### 流程图

```
每日 cron 触发（或用户需求）
    │
    ▼
main（布置任务，声明日报格式）
    │
    ▼
ResearchAgent 搜集情报
    │
    ▼
ContentAgent 生成日报
    │
    ├─→ 需要实时素材 ──→ ResearchAgent 补充
    │
    └─→ 自检
            │
            ▼
       提交发布
            │
            ▼
      main 汇报（如有异常）
```

### 特点

**日报是简单重复任务，不走完整审核流程：**
- ResearchAgent 搜集 → ContentAgent 生成 → 直接发布
- ReviewAgent 仅在异常时介入
- 自评分 < 7 才触发 ReviewAgent 审核

### 阶段1：任务布置

```markdown
## 雷达日报任务
**日期：** [YYYY-MM-DD]
**格式：** [标准格式]
**搜索范围：** [24小时]
```

---

## 三、运维故障处理工作流

### 核心角色

| 角色 | 职责 |
|------|------|
| **main** | 布置任务 + 重大决策 + 汇报用户 |
| **OpsAgent** | 故障检测 + 初步排查 + 执行修复 |
| **ResearchAgent** | 查官方文档 + GitHub issues + 最佳实践 |

### 流程图

```
故障触发（cron/heartbeat/用户报告）
    │
    ▼
OpsAgent 初步诊断
    │
    ├─→ 已知问题 ───→ 自动修复 ───→ 记录 ───→ main 汇报
    │
    ├─→ 未知问题 ───→ 升级 ResearchAgent
    │                      │
    │                      ▼
    │               ResearchAgent 调研
    │                      │
    │     ┌────────────────┴────────────────┐
    │     ▼                                 ▼
    │ 找到方案                          未找到方案
    │     │                                 │
    │     ▼                                 ▼
    │ OpsAgent 修复                    升级 main 决策
    │     │                                 │
    │     └─────────────┬───────────────────┘
    │                   ▼
    │              main 汇报用户
    │              用户决策
    │                   │
    │         ┌─────────┴─────────┐
    │         ▼                   ▼
    │      继续修复             暂停
    │         │                   │
    │         ▼                   ▼
    │    OpsAgent 执行        记录并汇报
    │         │
    │         ▼
    │    修复完成
    │         │
    │         ▼
    │    main 汇报用户
```
