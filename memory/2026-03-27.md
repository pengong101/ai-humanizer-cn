### 2. 雷达日报搜索修复（重大问题）
**问题：** 搜索返回0条/质量极差
**根因：**
- SearXNG 的 httpx 不读 HTTP_PROXY 环境变量，必须在 settings.yml 的 outgoing.proxies 中显式配置
- SearXNG JSON API 默认禁用，需在 search.formats 添加 json
- mihomo HTTP 代理：对外大部分国际站点（Google/DuckDuckGo/Wikipedia）返回 "access denied" 或 timeout
- settings.yml 是目录格式，需容器初始化后用文件覆盖

**最终配置：**
- SearXNG: openclaw_openclaw-net + HTTP_PROXY 环境变量 + settings.yml proxies 配置
- 可用引擎：bing ✅ (10 results)、sogou ✅ (10 results)、duckduckgo ✅ (10 results)
- ❌ google: access denied
- ❌ wikipedia/wikidata/qwant/startpage: timeout
- 备份配置：/root/searxng-settings.yml

**日报脚本修复：**
- SEARCH_TIME_RANGE: "day" → "week"
- QUALITY_THRESHOLD: -3 → -6
- MAX_PER_CATEGORY: 8 → 10
- SEARCH_ENGINES: ["bing", "sogou", "duckduckgo"]
- timeout: 15 → (10, 60) 自适应

**日报效果：** 23-26条，质量大幅提升

### 3. TUN 模式调试（失败）
**目标：** 让 mihomo 以 TUN 模式运行，实现全局代理
**失败原因：**
- system stack: "operation not permitted"（内核权限不足）
- gvisor stack: TUN 设备创建成功，但 auto-route 接管所有流量导致回路死锁
- mihomo 需要在 settings.yml 的 tun.dns-hijack 中使用 URL 格式（如 https://dns.google/dns-query），不能用裸 IP
- exclude-cidr 可避免回路，但需要代理服务器真实 IP
- 每次调试失败都导致服务器网络瘫痪（exec 全部超时）

**结论：** TUN 模式在此虚拟化环境不可行，HTTP 代理方案已够用
