## 💡 经验教训

### SearXNG Docker 部署教训
1. **VOLUME 声明覆盖 bind mount**: Dockerfile 的 VOLUME 声明会导致 `-v host:container` 失效
2. **IPv6 监听问题**: SearXNG 默认绑定 IPv6 `::`，Docker 端口映射对 IPv6 内部无效
3. **GRANIAN_HOST 环境变量**: SearXNG 使用 Granian ASGI server，`GRANIAN_HOST=0.0.0.0` 可强制 IPv4 监听
4. **代理配置要谨慎**: `outgoing.proxies` 全局代理会导致容器启动时崩溃（无法连接宿主机 7890）
5. **403 根因**: 不是 IP 绑定问题，是 SearXNG 应用层 `trust_localhost` / `trusted_networks` 配置

- MiniMax `image-01` 生成的图片文字会乱码，prompt 中禁止包含 text/label 等描述
