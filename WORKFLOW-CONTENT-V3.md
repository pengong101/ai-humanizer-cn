# 内容创作工作流 v3.0

**日期：** 2026-04-03  
**状态：** 基于用户反馈重新设计

---

## 核心改进

### 问题：v2.0 星型结构的缺陷

```
v2.0（星型）：
main ────→ ResearchAgent ────→ main ────→ ContentAgent ────→ main ────→ ReviewAgent
              ↑                                              ↑
              └──────────────────────────────────────────────┘
                        （验证阶段才连接，缺乏迭代）

问题：
1. main 成为瓶颈（每次都要经过）
2. Agent 间缺乏直接协作
3. 迭代效率低
4. 交接材料不明确
```

### 解决方案：协作 + 迭代模式

```
用户（你）
    │
    ▼
main（布置任务）
    │
    ├──────────────────────────────────────────────────────────┐
    │                                                          │
    ▼                                                          ▼
ResearchAgent ◄──────────────────► ContentAgent              main
  调研                                      创作                │
  提供素材 ──────────────────────────→ 申请更多素材             │
  反馈创作建议 ◄──────────────────────── 接受建议并迭代         │
                                                              │
                                            │                 │
                                            ▼                 │
                                      ReviewAgent              │
                                         │                    │
                    ┌────────────────────┼────────────────────┘
                    ▼                    ▼
               通过                    不通过
                │                      │
                ▼                      ▼
           发布                    反馈修改建议
                                    │
                                    ▼
                              迭代循环（最多3轮）
                                    │
                              3轮未过 → main 决策
```

---

## 一、工作流程详细设计

### 角色职责

| 角色 | 职责 |
|------|------|
| **main** | 布置任务 + 中期决策（必要时）+ 最终汇报用户 |
| **ResearchAgent** | 调研 + 提供素材 + 根据反馈调整研究重点 |
| **ContentAgent** | 创作主体 + 申请素材 + 接受反馈迭代 + 生成图片 |
| **ReviewAgent** | 质量审核 + 反馈修改建议 |

---

### 阶段1：任务布置（main）

**main 接收用户需求后，布置任务：**

```
任务布置：
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
主题：[用户需求描述]
目标平台：[公众号/小红书/其他]
字数要求：[X-Y字]
截止时间：[如有]
特殊要求：[如有]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**main 将任务转给 ResearchAgent + ContentAgent，声明进入协作模式**

---

### 阶段2：协作创作（ResearchAgent ↔ ContentAgent）

这是核心阶段。两个 Agent 直接协作、迭代，无需经过 main。

#### 2.1 初始化（ResearchAgent 发起）

**ResearchAgent 主动输出：**

```markdown
## 创作素材包 v1.0
**主题：** [主题]
**日期：** [日期]

### 核心数据点
1. [数据1] - [来源]
2. [数据2] - [来源]
3. [数据3] - [来源]

### 关键概念
- [术语1]：[解释]
- [术语2]：[解释]

### 配图建议
| # | 内容 | 关键词 | 来源 |
|---|------|--------|------|
| 1 | [封面图] | [关键词] | [NASA/EHT/其他] |
| 2 | [示意图] | [关键词] | [来源] |

### 文章角度建议
[建议3个不同角度，供 ContentAgent 选择]

### 置信度
[ResearchAgent 自评：X/10]
```

#### 2.2 ContentAgent 创作 + 迭代

**ContentAgent 收到素材包后：**

1. 选择角度
2. 撰写草稿
3. 标记需要补充的点
4. 随时向 ResearchAgent 申请素材

**ContentAgent 可主动发起：**

```markdown
## 素材申请
**申请编号：** RQ-001
**申请类型：** [数据/概念解释/配图/其他]
**需求描述：** [具体说明]
**紧急度：** [高/中/低]
```

**ResearchAgent 响应：**

```markdown
## 素材响应 RQ-001
**状态：** [提供/无法提供/部分提供]
**素材：**
[具体内容或说明原因]
**置信度：** [X/10]
```

#### 2.3 迭代直到自评分达标

**迭代规则：**
- ContentAgent 自评分 ≥ 7，且 ResearchAgent 确认素材完整 → 提交 ReviewAgent
- 自评分 < 7 → 继续迭代（向 ResearchAgent 申请更多素材）
- ResearchAgent 可主动提供新发现

**自评分标准（ContentAgent）：**
```
自检清单：
□ 事实准确（ResearchAgent 验证过）
□ 逻辑通顺（至少通读3遍无误）
□ 角度有趣（不是泛泛而谈）
□ 配图到位（封面+内图齐全）
□ 字数达标（[X-Y]字）

自评：[X/10]
是否提交：□ 是 □ 否
```

### 阶段3：提交审核

**ContentAgent 提交时提供：**

```markdown
## 作品提交 v1.0
**主题：** [主题]
**字数：** [实际字数]
**平台：** [公众号/小红书]
**创作时长：** [X分钟]

### 文章正文
[完整文章]

### 配图清单
| # | 文件 | 内容 | 来源 |
|---|------|------|------|
| 1 | cover.jpg | 封面 | NASA |
| 2 | fig1.jpg | 示意图 | 自制 |

### 自评分
[ContentAgent 自评：X/10]

### 提交声明
本作品已完成自检，事实准确性由 ResearchAgent 背书。
```

---

### 阶段4：ReviewAgent 审核

**ReviewAgent 输出：**

```markdown
## 审核报告 v1.0
**作品：** [主题]
**审核时间：** [日期]

### 综合评分
[X/10]

### 问题列表
| # | 位置 | 问题 | 严重程度 | 修改建议 |
|---|------|------|---------|---------|
| 1 | 第二段 | 数据过时 | 高 | 更新为[新数据] |
| 2 | 配图 | 说明不准确 | 中 | 改为[说明] |

### 判定
- [ ] 通过（≥7分）
- [x] 不通过（<7分），需修改后重新提交
- [ ] 条件通过（小修改后可发布）

### 审核轮次
第 1/3 轮
```

---

### 阶段5：迭代修改

**不通过时：**

```
ReviewAgent ──→ ContentAgent + ResearchAgent
                    │ 收到修改建议
                    ▼
              讨论并修改
                    │
                    ├─→ ResearchAgent 补充素材（如需要）
                    │
                    └─→ ContentAgent 修改文章
                                │
                                ▼
                          提交 v2.0
                                │
                                ▼
                          ReviewAgent 再次审核
                                    │
                          ┌─────────┴─────────┐
                          ▼                   ▼
                      通过                 不通过（继续迭代）
                          │                   │
                    第2/3轮              第3/3轮
```

---

### 阶段6：升级决策（如3轮未通过）

**3轮审核未通过时：**

```markdown
## 升级报告
**作品：** [主题]
**审核轮次：** 3轮未通过
**最后得分：** [X/10]

### 问题汇总
1. [问题1] - 持续存在
2. [问题2] - 部分改善
3. [问题3] - 新出现

### 建议选项
A. 暂停任务 - [原因]
B. 继续迭代 - [改进方向]
C. 调整要求 - [如降低字数、换角度]

### main 决策
[等待 main（CEO）给出方向]
```

**main 决策后：**
- 继续 → 返回阶段2（ResearchAgent ↔ ContentAgent）
- 暂停 → 告知用户，任务暂停
- 调整 → 更新任务要求，重新评估

---

### 阶段7：发布 + 汇报

**ReviewAgent 通过后：**

```markdown
## 发布审批
**作品：** [主题]
**最终评分：** [X/10]
**发布平台：** [公众号/小红书]

[文章预览]

按要求发布？[确认/取消]
```

**用户确认 → ContentAgent 执行发布 → main 向用户汇报完成**

## 三、Agent 协作约定

### 主动沟通
- ResearchAgent 完成素材包后**主动通知** ContentAgent
- ContentAgent 需要素材时**主动申请**，不等待
- ResearchAgent 发现新素材时**主动推送**给 ContentAgent

### 响应时效
- 素材申请：ResearchAgent 应在[合理时间]内响应
- 紧急申请（高紧急度）：[快速响应]

### 争议解决
- ResearchAgent 和 ContentAgent 对素材有争议 → 提交 ReviewAgent 裁决
- ReviewAgent 和 ContentAgent 对修改有争议 → 提交 main 决策

### 记录要求
- 每个版本（素材包v1.0/作品v2.0等）需保存
- 最终版本标注 `[FINAL]`
- 方便追溯和审计

## 五、main 的角色

### 布置任务
- 明确主题、要求、截止时间
- 声明进入协作模式

### 中期介入（仅必要时）
- 3轮审核未通过时的决策
- Agent 间争议升级时的裁决
- 用户需求变更

### 不介入
- 创作过程（Agent 自主协作）
- 素材选择（Agent 间协商）
- 修改细节（ReviewAgent 给出建议，ContentAgent 执行）

### 最终汇报
- 任务完成 → 向用户汇报结果
- 任务暂停 → 向用户说明原因，请求指示
- 任务取消 → 向用户确认
