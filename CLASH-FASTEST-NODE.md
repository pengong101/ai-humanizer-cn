# ⚡ Clash 最快节点自动选择配置

**更新时间：** 2026-03-11 14:32  
**功能：** 自动测速 + 选择最快节点  
**测速间隔：** 5 分钟

---

## 🎯 核心功能

### 自动测速

**配置：**
```yaml
- name: '⚡最快节点'
  type: url-test
  interval: 300  # 每 5 分钟测速一次
  tolerance: 50  # 容差 50ms
  url: 'http://www.gstatic.com/generate_204'
```

**工作原理：**
1. 每 5 分钟对所有节点测速
2. 选择延迟最低的节点
3. 自动切换（延迟差>50ms 时）

---

**每组独立测速：**
- 测速间隔：5 分钟

## 📊 代理组层级

```
默认规则 → ⚡最快节点（自动测速）
    ↓
手动选择 → 隧道加速（可手动切换）
    ↓
备用方案 → 自动选择（24 小时测速）
    ↓
故障转移 → 故障转移（2 小时测速）
```

## 🔧 配置说明

### 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| interval | 300 | 测速间隔（秒） |
| tolerance | 50 | 切换容差（毫秒） |
| url | gstatic.com | 测速目标 |

### 测速逻辑

**自动切换条件：**
```
当前节点延迟 - 最快节点延迟 > tolerance(50ms)
→ 自动切换到最快节点
```

## 🎯 使用方式

### 默认使用

**所有流量自动走最快节点：**
```yaml
rules:
  - 'MATCH,⚡最快节点'
```

### 手动切换

**需要手动切换时：**
1. 访问 Clash Dashboard
2. 选择"隧道加速"代理组
3. 手动选择节点

## 🚨 故障排查

### 问题 1：节点不切换

**检查：**
```bash
# 查看当前延迟
curl http://127.0.0.1:9090/proxies/⚡最快节点

# 查看测速日志
docker logs clash | grep "delay"
```

**解决：**
- 等待下次测速（5 分钟内）
- 手动切换节点

---

### 问题 2：测速失败

**检查：**
```bash
# 测试测速 URL
curl --proxy http://192.168.1.122:7890 \
  http://www.gstatic.com/generate_204
```

**解决：**
- 检查 gstatic.com 可访问性
- 更换测速 URL
